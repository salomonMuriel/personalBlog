Vibe-coding ya no tan a la carrera - Claude Code bien usado 🔥

Después de la hackathon aprendí de primera mano que si y que no hacer al usar Claude Code.

Para Ignia estamos construyendo tech que lleve la experiencia de nuestros programas (y los de nuestros aliados como [Sistema B](https://www.linkedin.com/company/sistema-b/)... y otro que anunciamos esta semana 😏 ) a otro nivel.

Estamos construyendo a Ignacio 🔥 , un agente que pueda 'mentorear' en tiempo real a los participantes de un programa.

Que puedan interactuar con él por web o WhatsApp, que tenga contexto de sus proyectos y que pueda dar consejo experto según los problemas que tengan.

Con Claude Code, en ~4 sesiones de ~3 horas cada una he hecho lo que antes me habría tomado un mes (mio y de un dev junior!).

Tips:
1️⃣  Tener una carpeta .claude con archivos de contexto. Yo le tengo [CLAUDE.md](http://CLAUDE.md), [SPECS.md](http://SPECS.md) y [PLAN.md](http://PLAN.md) - para darle contexto del proyecto, de las especificaciones técnicas y del plan de trabajo.
2️⃣  De primeritas antes que nada, escribir el [CLAUDE.md](http://CLAUDE.md) (y el [SPECS.md](http://SPECS.md), si ya sabes que tech quieres usar) muy detallados y darle "ultrathink an implementation plan" que meta al [PLAN.md](http://PLAN.md). Editarle el resultado con juicio.
3️⃣  Solo construir una cosa a la vez. Apenas termine, /clear y seguir con lo próximo.
4️⃣ Usar Plan Mode un montón. Cuando uno este contento con el plan si dejarlo trabajar.

Va el master Ignacio. Hecha con Nano Banana y animado con Flow/Veo3.

Alguien tiene más tips? A alguien le da curiosidad de como usar Claude Code? Avisenme en los comentarios o por interno y vemos si cuadramos una sesión o taller.

PD: Decidí usar el Agents SDK de OpenAI y está increíble. Lástima no lo había visto para la hackathon.
